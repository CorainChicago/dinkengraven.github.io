Arrays & Hashes in Ruby

August 14, 2015

To the budding Rubyist, arrays and hashes may appear to be the same thing (or very close!). While they do serve similar functions, there are cases wherein you may want to use one over the other. Here is a brief review.

First, there are several things that arrays and hashes have in common:

  • Both store information.
  • This information is stored in an indexed format.
    • Each item is stored in a specific place, and items are stored in a specific order.
    • You can access any item within the array or hash by accessing its index.
    • Indices use "zero-based numbering," so the first item in an array or hash is at index 0, the second is at index 1, the third is at index 2, and so on.
  • Both hashes and arrays are good for storing collections of information (multiple items).
  • Both can store any type of object (integers, strings, Boolean values, other arrays, hashes, etc.). Note: In hashes, the key AND value can each be any type of object.

Now let's see what sets arrays and hashes apart.

And you may want to use a hash if...

  • You want a very specific 1:1 relationship between two pieces of information.
    • 
                dogs = { "Maggie" => "Pointer/Cattle Dog",
                "Sherman" => "Shepherd/Husky",
                "Pearl" => "Newfoundland"
                }
    Here we have a hash that uses the dog's name as the key, and the breed as the value. Accessing any dog name will return the dog's breed.

  • If you don't know the index of an element, but you do know the key. It's much faster/easier for the computer to search through a hash for "Maggie" and return "Pointer/Cattle Dog" than it is for it to search an entire array for that information.
    • 
                dogs["Maggie"]      # > "Pointer/Cattle Dog"

You may want to use an array if...

  • You don't particularly need any one piece of information attached to another.
    • 
                cats = ["Egypt", "Beast Boy", "Gorby Thunderhorse"]
    (I know, I know, there are different breeds of cats. But come on, no one references cat breeds as often as they do dog breeds! The lesson here is that dogs are better.)

  • You want to be able to quickly access, remove, or add individual elements.
    • 
                cats[0]             # > "Egypt"

                cats.pop            # > "Gorby Thunderhorse"
                cats.inspect        # > ["Egypt", "Beast Boy"]

                cats.push("Henry")
                cats.inspect        # > ["Egypt", "Beast Boy", "Henry"]

You could also think of it this way: An array is a box with dividers, and a hash is a set of small drawers (like a card catalog). You can carefully arrange items between the dividers in the array, or you can assign a name to each drawer in a hash and store one value within it. Perhaps the best way to test this out is to use irb within your terminal! Try setting up an array and a hash, and accessing information in each one.
